Naive implementation

And voila ... i only implemented 3 pathes , just to see what happens...

In terminology i'll use :
  • the grid is where we draw the lines. It has rows and columns.
  • pathes as the lines ...
  • step as a point ( a path step ) and steps as a grid row ( or pathes steps ) .
  • left and right border of the grid : the path should stay inside the grid...

First we pick up a random number which corresponds to a grid column index. Then for each step a path has choice between going left, straight , or right . there are 2 constraints :
  • when a path is on the left border, it has only two choices : going straight or right
  • when a path is on the right border , it has only two choices : going straight or left

Inside the js - i talk only about the the algo here .
  • the pathes are stored in a 2D array. pathes[grid_row_index][path_number] = grid_column_index
  • by using the grid column indexes for the steps, we can make a random between -1 ,0 and 1 in order to know the next path direction and step location.
    • -1 would be going on the left
    • 0 would be going straight
    • 1 would be going on the right
  • the heart of the algorithm is inside the NEXT() function

Some issues whith this naive implementation : ( try to refresh browser if it does not appear ... ) :
  • pathes can cross each other.
  • pathes can be extremely straight . or we can imagine they overlap on 3 or 4 steps , which might not be so nice .
