Naive implementation

here, condider as before that all the pathes are on a grid... for testing , i only implemented 3 pathes , to see what happens...
- the grid is defined by rows ( or steps ) , and columns
- at step 1 ( or row 1 ) we randomly pick up for each path a column
- for the next steps , for each pathes, we randomly choose if it goes on the left, on the right, or it goes straight.
- watch out as if the path is on the border of the grid , there are only two choices.. eg if it is on the left of the grif the path can go straight, or on the right only...


Some issues whith this naive implementation : ( try refresh browser if it does not appear ... ) :
- pathes can cross each other.
- pathes can be extremely straight . or we can imagine they overlap on 3 or 4 steps , which might not be so nice .
