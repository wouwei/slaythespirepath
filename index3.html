no crossing!

The changes :
  • we get a random number of pathes from 2 to 4
  • we keep the structure pathes[row][myPathNumber] = column
  • we add this constraint for each row : pathes[n][0] < pathes[n][1] < pathes[n][2]< etc ...
  • which means the path 0 can never cross path 1 , which can never cross path 2 , etc..


As for the algorithm :
  • As before , a path has 3 choices , except when it is :
    • on its left neightbour step : the path then can only take its left neighbour step or go on the right
    • on its right neighbour step : the path then can only take its right neightbour step or go on the left
    • on its left and right neighbour step : the path then can only go straight...
  • we only replaced here the grid border by the path neighbour steps.


As we calculate the next steps for the pathes from the left path to the right path , 2 issues raised :
  • the left issue :
    • the path is on the same step as its left neighbour.
    • we calculated that the left neighbour goes on the right
    • the path has only one choice : going right
  • the border grid :
    • the path is on the right border of the grid .
    • we calculated that the left neighbour goes on the right
    • the path has only one choice : going straight .
