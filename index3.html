Not so naive implementation

First thing changed, we have randomly from 2 to 4 pathes. The concept here is to have pathes which cannot cross each others

At the start of the Algorithm :
- like last time, we create a 2D array called pathes, but nesting now from 2 to (columns/2) pathes

- before initializing the pathes with the first steps , it is important to be sure that the first path is on the left of the second which should be on the left of the next paths, etc..
- said differently. we have pathes[row][myPathNumber] = column . row is the row on the grid, column : column on the grid, myPathNumber : mypath from the left to the right on the grid - that said, we want to respect pathes[n][0] < pathes[n][1] < pathes[n][2]< etc ...



At the heart of the algorithm :
- we want to respect pathes[n][0] < pathes[n][1] < pathes[n][2]< etc ...
- we calculate steps from the left path to the right path ..
- if we take a path to calculate its next step, we need to take into account , the already calculated step from the left path, and the actual step from the right path.
- the NEXT function has been modified to take into account its neighbours.
  • we take the leftStep, which is the calculated step of the left neighbour of the path. if no left neighbour, we take the left side of the grid as a limit
  • we take the rightStep, which is the actual not-yet calculated step of the right neighbour of the path.if no right neighbour, we take the the right side of the grid as a limit.
  • we the randomly calculate the next step of the selected path, in the boundary of its neighbours...
-two special cases has been raised :
  • left path issue..
    1. the selected path and its left neighbour are on a same point .
    2. the left path went on the right ( remember, steps calculus goes from left path to right path ) .
    3. the selected path goes awol OUTSIDE the left / right pathes boundaries...
    4. selected path needs to follow its left neighbour, in order to avoid a crossing...
  • The right grid border
    1. selected path is on the right grid border.
    2. its left neighbour went on right, on the right grid border
    3. As in the algorithm, we consider that if the left path will share a point with the selected path, the selected path can go only straight , or on the right..
    4. .. We need for the selected path to not go outside the grid.. And can only go straight in the case
- a new function has been added to handle the calculation of all the pathes steps, called NEXTSTEPS...
